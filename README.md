# Data Science Course — Локальный самоучитель

Полноценный локальный курс-самоучитель по Data Science: Python, SQL, NumPy, Pandas, ML, визуализация, статистика, проекты и подготовка к собеседованиям.

Запускается **одной командой** — `python run.py`. Браузер открывается автоматически.

## Стек

- **Backend:** Python 3.11+ + FastAPI + SQLite (stdlib)
- **Frontend:** Vanilla HTML + CSS + JavaScript (без фреймворков)
- **Python sandbox:** Pyodide (Python в браузере через WebAssembly) — локально в `frontend/vendor/`
- **SQL sandbox:** sql.js (SQLite в браузере) — локально в `frontend/vendor/`

Никакого Node.js, npm, Docker или облака. После первой установки интернет не нужен.

## Установка и запуск

### Шаг 1. Убедись, что есть Python 3.11+

```bash
python --version
```

### Шаг 2. Запусти

```bash
cd ds_course
python run.py
```

`run.py` автоматически:
1. Установит зависимости (`fastapi`, `uvicorn`, `pydantic`) — нужно только при первом запуске
2. Скачает Pyodide (~10 МБ), sql.js (700 КБ) и wheel-файлы для `numpy`, `pandas`, `matplotlib`, `scikit-learn`, `scipy` + все зависимости (~80 МБ) в `frontend/vendor/` — нужно только при первом запуске
3. Инициализирует SQLite БД с контентом (130 уроков, ~1010 упражнений)
4. Запустит FastAPI сервер на `http://localhost:8000`
5. Откроет браузер автоматически

### Альтернативно (если уже есть зависимости)

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

И открой `http://localhost:8000` в браузере.

Два полноценных проекта в формате **интерактивного чата-комикса** (15 шагов каждый):

| Проект | Тема | Коллеги |
|--------|------|---------|
| **Space** | Анализ миссий NASA | Научная Скалли, Мрачный Малдер, Озабоченный Дима, Весёлый Лёха |
| **Gaming** | Анализ поведения игроков | Продуктивная Галя, Серёжа-геймер, Дизайнер Стёпка, Мрачный Игорь, Странный Витя |

**Фишки чата:**
- Босс (клиент) появляется только в шаге 1 (даёт задачу) и 15 (принимает работу)
- Ты — новичок: в шаге 1 пишешь «Хорошо, сделаю», потом паникуешь и просишь помощи
- Коллеги отвечают в чате (советы + шутки в характерах)
- **Анимация печатания** — пузырёк с точками, которые плавно появляются/исчезают (как в Telegram)
- **Строгий порядок сверху вниз** — сообщения раскрываются последовательно, не прыгают
- **Клик по аватарке** — открывается модалка с фото 128×128, должностью и описанием персонажа
- **Эмодзи-реакции** под сообщениями коллег (👍 😂 🔥 💩 🤡) — клик чтобы поставить/убрать/переключить
- Прогресс сохраняется в `localStorage` (текущий шаг, подсказки), кнопка сброса в заголовке

## Использование

### На уроке

1. **Читай теорию** — сворачивай/разворачивай секции кликом.
2. **Запускай примеры** — в редакторе уже есть starter code, нажми ▶ Запустить.
3. **Решай упражнения:**
   - ✏️ Отредактируй код в редакторе
   - ▶ **Запустить** — выполнить код и увидеть вывод
   - ✓ **Проверить** — прогнать тест-кейсы и узнать, решена ли задача
   - 💡 **Подсказка** — показать подсказку
   - 📋 **Решение** — загрузить эталонное решение в редактор
   - ↺ **Сбросить** — вернуть starter code
4. **Отмечай урок пройденным** кнопкой вверху страницы.

### SQL упражнения

В каждом SQL-упражнении уже загружена стандартная схема:
- `players (id, name, score, team_id)` — 5 игроков
- `teams (id, team_name)` — 3 команды

Пиши SELECT-запрос, нажми ▶ Запустить чтобы увидеть результат-таблицу, или ✓ Проверить для проверки.

### Прогресс

Прогресс сохраняется в `localStorage` браузера и в `SQLite` (`user_progress` + `exercise_attempts`).

Чтобы сбросить прогресс: открой DevTools → Application → Local Storage → удали ключи `ds_completed_lessons` и `ds_solved_exercises`.

## Технологические решения

### Почему Pyodide и sql.js в браузере?

Безопасность и простота: код пользователя выполняется изолированно, сервер только отдаёт контент. Это значит, что серверная часть тривиальна — никаких subprocess для запуска пользовательского кода, никаких таймаутов и рисков.

### Почему SQLite, а не PostgreSQL?

- Один файл БД — легко бэкапить и удалять
- Нет отдельного сервиса
- Достаточно для всех запросов курса
- Легко сбросить (просто удалить `course.db`)

### Почему без React/Vue?

Минимизация зависимостей: один файл `app.js` управляет всем. Никаких сборок, транспайлеров, npm install.

## Расширение контента

### Добавить упражнение в урок 1.1

Открой `app/seed_block1_part1.py`, найди функцию `_1_1()`, в массив `exercises` добавь:

```python
ex(N, "python", "Текст задания", "starter code", "solution code",
   [{"check": "isinstance(x, int)", "msg": "Описание ошибки"}],
   ["Подсказка 1", "Подсказка 2"], difficulty),
```

Удали `course.db` и перезапусти — упражнение появится.

### Добавить урок в блок 10 или 11

Открой `app/seed_augmented_content.py`, создай фабричную функцию вида `_10_1()` или `_11_4()`, верни словарь с ключами `title`, `content_json` (список секций), `exercises` и `difficulty`. Добавь её в `LESSONS_B10` или `LESSONS_B11` в конце файла — seed подхватит автоматически.

Для миграции существующей БД открой `app/migrations.py`, добавь в `run_migrations()` вызов `_add_new_lessons(...)` с новыми уроками.

### Обновить диалоги финальных проектов

Открой `app/seed_capstone.py`, измени `SPACE_STEPS` или `GAMING_STEPS` — массив шагов с полями `dialogue` (массив реплик клиент/я) и `task`.

### Добавить проект

Открой `app/seed_meta.py`, в список `PROJECTS` добавь словарь.
